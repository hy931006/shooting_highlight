# 投篮检测优化方案

> **设计日期**: 2026-03-20
> **问题**: 用户视频是进球集锦（约250次投篮基本都进），但当前算法只检测到44进球vs261丢球

## 1. 问题分析

当前检测结果：
- 进球: 44次 (其中31次高置信度>0.8)
- 丢球: 261次 (全部低置信度)
- 有53次"丢球"有超过50个轨迹点，可能是误判

根本原因：
1. `hoop_proximity=50` 像素阈值太小
2. 进球判定要求轨迹精确经过 `hoop_x` 附近，过于严格

## 2. 优化方案

### 2.1 参数调整

| 参数 | 当前值 | 新值 | 说明 |
|------|--------|------|------|
| hoop_proximity | 50.0 | 80.0 | 增大篮筐附近判断范围 |

### 2.2 判定逻辑优化

**当前逻辑**:
1. 最高点在篮筐附近 (`abs(vertex_y - hoop_y) < hoop_proximity * 2`)
2. 后续存在点 `y > hoop_y` (向下穿过篮筐水平线)
3. 球在篮筐x坐标附近 (`abs(x - hoop_x) < hoop_proximity`)

**优化后**:
1. 最高点在篮筐附近 (`abs(vertex_y - hoop_y) < hoop_proximity * 2`) - 保持
2. 后续存在点 `y > hoop_y` (向下穿过篮筐水平线) - 保持
3. **放宽**: 只要下落阶段有任意一点在篮筐x的较宽范围内即可 (`abs(x - hoop_x) < hoop_proximity * 1.5`)

### 2.3 备选判定逻辑

增加一个辅助判定：当轨迹完整但未命中时，检查是否符合进球特征：
- 轨迹有明显的上升-下降模式 (a < 0)
- 最高点高度与篮筐接近
- 如果是，返回 "made" 类型

## 3. 实现计划

1. 修改 `TrajectoryAnalyzer.__init__` 默认参数
2. 修改 `analyze_trajectory` 中的判定逻辑
3. 重新测试并对比结果

## 4. 验收标准

- 对于进球集锦视频，检测准确率应显著提升
- 目标: 进球检测数应接近真实投篮数的70%以上
